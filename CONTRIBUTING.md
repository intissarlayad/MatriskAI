# Contributing to MatriskAI

First off, thanks for taking the time to contribute! ❤️

All types of contributions are encouraged and valued. See the Table of Contents for different ways to help and details about how this project handles them. Please make sure to read the relevant section before making your contribution.

## Conventional Commits

We follow the **Conventional Commits** specification. Please format your commit messages accordingly to keep the repository history clean and professional:

*   `feat:` A new feature (e.g., `feat: add AI prediction module`)
*   `fix:` A bug fix (e.g., `fix: resolve dashboard loading issue`)
*   `docs:` Documentation only changes
*   `style:` Changes that do not affect the meaning of the code
*   `refactor:` A code change that neither fixes a bug nor adds a feature
*   `perf:` A code change that improves performance
*   `test:` Adding missing tests or correcting existing tests

## Pull Request Process

1.  Fork the repository and create your branch from `main`.
2.  If you've added code that should be tested, ensure your code handles edge cases.
3.  If you've changed APIs or the Pipeline structure, update the documentation in `/docs` and the `README.md`.
4.  Ensure your code is clean and properly commented.
5.  Issue that pull request! Provide a clear description of the problem you are solving.

## Code of Conduct

Be respectful and considerate of others. This is an academic and professional project aimed at pushing the boundaries of AI in Supply Chain.
